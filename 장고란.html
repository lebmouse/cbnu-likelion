프시케 Jan 3, 2018


Python Django

Python Django(파이썬 장고/드잔고...)는 파이썬 기반의 무료 오픈소스 웹 애플리케이션 프레임워크(Open Source Web Application Framework) 이다.

여기서 오픈소스(Open Source)란 장고(Django)를 구성하는 코드들이 온라인상으로 언제든 참조할 수 있다는 것을 뜻하고, 웹 애플리케이션 프레임워크(Web Application Framework)란 웹 애플리케이션(우리가 흔히 아는 웹 페이지들)을 개발할 수 있도록 만들어 놓은 하나의 작업장이라고 생각하면 됩니다.

프레임워크(Framework)

그럼 왜 이런 프레임워크라는 작업장을 만들어 놓았을까요? 웹을 개발할 때 굉장히 빈번하게 반복적인 문제를 해결해야 될 때가 많습니다. 예를들면, 사용자들의 회원가입, 로그인, 로그아웃 등의 사용자 인증 및 관리 문제, 아니면 게시판과 같이 사용자들이 커뮤니케이션을 할 수 있는 공간을 마련하여 업로드, 다운로드를 할 수 있도록 하는 문제 등이 이에 해당합니다. 우리가 아는 거의 모든 웹 페이지들은 이러한 기능들을 기본적으로 제공하죠? 그렇다면 우리도 그럴싸한 웹을 서비스 하려면 기본적인 기능들을 제공해야 됩니다. 이러한 기능들을 맨땅에 헤딩하지 않도록 편리한 개발환경(예를 들면, 미리 이러한 기능들이 구현이 되어 있다던지 등…)을 만들어 놓은 것이 바로 프레임워크의 역할입니다.

웹(Web)

그렇다면 이제 프레임워크에서 웹을 프레임워크에서 어떻게 개발을 할까요? 그러면, 웹은 어떤식으로 작동을 하게 될까요? 프레임워크에서 어떻게 개발을 해나가야 되는지는 앞으로 계속 포스팅을 할 예정입다. 웹이 어떻게 작동하는지에 대해서만 조금 이야기를 하겠습니다. Client-Server Model
웹 사용자(Client)는 웹 브라우저(Web Browser, 예. Chrome, Internet Explorer, FireFox 등)를 보면서 여러가지 행동들을 하게됩니다. 예를 들면 검색을 하는 일이요. 사용자의 모든 행동들은 웹 브라우저를 통해 서버(Server)로 “나는 지금 어떤 행동을 하고 싶다.”를 편지의 형태로 보내게 됩니다. 이것을 요청(Request)라고 합니다. 그렇다면 서버(Server)는 사용자가 보낸 편지에 답장을 해줘야겠죠? 이것을 응답(Response)라고 합니다. 서버는 응답을 브라우저를 통해 보여주게 되고, 그렇게 되면 사용자는 그 브라우저를 통해 자신이 요청한 사항에 대하여 적절한 응답을 보게 되겠죠!
저희는 앞으로 그 웹 브라우저에 보이는 웹 페이지 또는 웹 애플리케이션을 제작해보도록 할 것입니다.

장고(Django)의 특징

장점

  • 비교적 쉬운 언어인 Python을 기반으로 두기 때문에 익히기 쉽다.
  • 익히기 쉬운 만큼 개발속도도 빠르고, 코드 완성도를 높게 유지할 수 있으며 확장성도 좋다.
  • 웹 애플리케이션이 제공되어야하는 사용자 인증, 사용자 관리 등을 기본적으로 기능 구현이 되어 있다.
  • 성공적인 도입 사례가 많다. (Instagram이 대표적인 사례다.)
  • 개발 비용을 크게 절감시킬 수 있다.

단점

  1. Python을 기반으로 하기 때문에 비교적 쉬운 언어이지만, Python이 객체 지향 프로그램이므로, 객체 지향 프로그램에 대한 이해도가 필요하다.
  2. Python의 단점을 대부분 포함한다.
  3. 성능이 다른 웹 애플리케이션 프레임워크(Node.js 등)보다 좋지 못하다.
  4. 한글화된 문서가 그렇게 많지 못하다.
  5. 웹 애플리케이션 프레임워크부터, 웹이 어떻게 작동하는지, 장고의 특징은 뭔지에 대해서 간략하게 알아 봤습니다. 이어지는 포스트 내용들은 장고를 통해 웹을 만들어 직접 서비스 해보는 것까지 이어 나가겠습니다!
